# What Is Fully Reproducible from the Anonymous Package

This file states exactly what the bundled supplement can reproduce on its own, what it only supports by inspection, and why the package is structured that way.

## 1. Included sample logs

The zip contains a reduced log subset with 36 episodes total:

- `baseline` - 3 episodes
- `single_norm` - 3 episodes
- `multi_free` - 3 episodes
- `multi_norm` - 3 episodes
- `solo_phi_exposed` - 3 episodes
- `homo_ds` - 3 episodes
- `multi_free_matched` - 3 episodes
- `multi_norm_matched` - 3 episodes
- `single_norm_matched` - 3 episodes
- `temp_high` - 3 episodes
- `multi_free_phi3_qwen3_ds_ollama` - 3 episodes
- `multi_norm_phi4_llama3_ds_ollama` - 3 episodes

These logs are included so reviewers can inspect the runtime format and rebuild a nontrivial subset of the pipeline without shipping the full corpus.

## 2. Fully reproducible from the bundled package alone

The following can be reproduced directly, without obtaining any additional logs or model weights:

### 2.1 Pipeline mechanics on the bundled subset

Running

```powershell
py -3.13 export_data.py
py -3.13 build_edtrack_repair_addendum.py
py -3.13 build_edtrack_appendix_tables.py
py -3.13 build_response_region_map.py
```

will successfully rebuild outputs from the files actually present in the package.

### 2.2 Deterministic appendix/reference artifacts

The following are fully reproducible from the packaged files:

- `results/nonllm_reference_policies.csv`
- `results/nonllm_reference_policies.md`

These are generated by simulation inside `build_edtrack_appendix_tables.py` and do not depend on missing full-corpus logs.

The auxiliary and effect-size table builders also run on the bundled subset, but their exact paper values require the full log corpus. The package therefore bundles their canonical full-corpus exports for inspection.

### 2.3 Figure regeneration from bundled exports

The response-label figure can be regenerated directly from the bundled `results/experiment_summary.csv`:

- `results/response_region_map.png`
- `results/response_region_map.pdf`

## 3. Inspectable but not fully rebuildable from sample logs alone

The following paper artifacts are bundled in their exact exported form, but cannot be rebuilt exactly from the 36 sample episodes alone:

- `results/experiment_summary.csv`
- `results/robustness_bootstrap_ci.csv`
- `results/edtrack_repair_addendum.csv`
- `results/edtrack_repair_addendum.md`
- `results/commons_transfer_summary.csv`
- `results/commons_transfer_summary.md`
- `results/auxiliary_metrics_summary.csv`
- `results/auxiliary_metrics_summary.md`
- `results/effect_size_summary.csv`
- `results/effect_size_summary.md`
- `results/response_framework_overview.png`
- `results/response_framework_overview.pdf`
- `results/mechanism_phi4_overview.png`

Reason:

- the paper reports a corrected full corpus with `85 conditions`, `8,500 episodes`, `16,900 agent trajectories`, and `507,000 logged actions`
- the anonymous zip intentionally contains only a reviewer-sized log slice
- re-running `export_data.py` on the bundled logs will regenerate a valid **subset** export, not the full corpus export used in the paper

This is why the package includes both:

- exact exported paper-side CSV/MD/PDF outputs for inspection
- a smaller log slice for pipeline transparency

## 4. Requires local model access or non-bundled assets

The following are **not** fully reproducible from the zip alone:

### 4.1 Full-corpus behavior results

Rebuilding the exact published 85-condition corpus requires:

- the full local log set, or
- re-running the benchmark with local access to the specified `llama.cpp` and Ollama model stack

### 4.2 GPU mechanism follow-up

Re-running the Phi mechanism probe requires:

- the separate GPU environment `.venv-mech312`
- CUDA-capable hardware
- local access to the upstream Phi Hugging Face weights

The package includes the exported mechanism outputs so that reviewers can inspect the evidence even if they do not rerun the probe.

## 5. Why the package is structured this way

The package is designed for anonymous review. It is intentionally small:

- zip size: about `1.54 MB`
- no model weights
- no full log dump

The purpose is to let reviewers inspect:

- the exact evaluation code path
- the exact exported paper numbers
- a concrete sample of raw episode artifacts
- the figure/table builders

without forcing submission-time redistribution of a much larger local corpus or third-party model weights.

## 6. Practical reading guide for reviewers

If you want to verify the paper's claims with minimal effort:

1. inspect the bundled canonical outputs in `results/`
2. inspect the sample raw episodes in `logs/`
3. run the builders to confirm the pipeline works on the subset

If you want exact end-to-end regeneration of the published corpus, that requires the full local logs or a local rerun with the model stack described in `paper/ASSETS_AND_LICENSES.md`.
